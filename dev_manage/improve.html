# 软件工程师的自我修养

作为一位软件开发工程师，思考如何提升自己的专业技能。

  1. 选择永远比努力重要；
  2. 所有好的代码都不是一次性写出来的，而是通过不断修改完善优化出来的；
  3. 如果自己不能改变现状，那就让环境来改变自己；
  4. 你的理解不是一蹴而就的，它在不断地追求过程中完善。——许钧
  5. 接口除了定义方法外，还可以定义常量，可以将写两种结合起来，作为注解路径火参数，供后续调用；

问题

  1. 代码怎么写才能更简洁高效？
  2. 系统该怎样改进，性能才能更优异？
  3. 界面交互该怎么设计，更能提升用户体验？

# 软件工程师招聘面试题

  • Java基础部分

    • Java集合类
    • 多线程
    • 网络编程（Socket、NIO、BIO、Netty）
    • Spring
    • SpringMVC（常用注解、注解原理）
    • SpringBoot

  • MySQL

    • SQL优化
    • 索引数据结构

  • 缓存队列

    • Redis（数据结构类型、）
    • RabbitMQ
    • Kafka

  • 前端部分

    • JS、VUE
    • 前后端请求过程（GET/POST区别）
    • DOM事件传播机制

  • 操作系统

    • Linux(Shell脚本)
    • Git/SVN
    • Docker容器

  • 业务部分

    • 谈谈自己对做过项目的看法？

  • 期望薪资

招聘结果打分

  1. 技术得分（0~25分）
  2. 态度得分（0~25分）
  3. 业务得分（0~25分）
  4. 期望薪资得分（0~25分）

钛星数安

面试时间：2020-06-01 下午两点半 面试结果总结：这次是一场测试面试，路程比较远，有一个半小时的路程。面试结果比较理想，要了25K，别人也没讨价还价。但是综合来看，还是不大想去。 主要原因是比现在环境也高不了多少，日常自由要受到限制的更多。而且去开发前端一些的软件，本身产品对我来说，吸引力并不是多大。这次去还有一个 目的，是检测自己当前的技术实力水平。总共有三个人连续面，一个接一个，第一个面试人的水平还行，他问了一些业务知识和Java相关的知识点，业务知识我回答的 很好，但是技术方面的知识我没能答上来，这个我自己明白，知识还是需要巩固复习的，不然靠吃老本，最终白跑。第二个面试只能算是小孩。第三个 应该是他们的技术主管，业务水平比较深。这基本就是这次面试的主要内容。

第一个面试提问的几个没有答上来或没答好的问题：（1）Lock原理；（2）CAS原理；（3）Java垃圾回收机制；（4）Security与Shiro框架；
